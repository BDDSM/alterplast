История версий

1.1.0.1
[+] начало построения объектной модели классов 1С++ - например, можно изучать в VBA из MS Office.
[+] новые COM-объекты CoClasses и CoClassInfo для использования в скриптах
[+] новые COM-объекты CoDefFiles и CoDefFile для использования в скриптах
[+] COM-объекты CoClassss, CoDefFiles умеют использовать индекс по умолчанию;
[+] COM-объекты CoClasses, CoDefFiles можно использовать с оператором For Each из Visual Basic;
[+] новый метод плагина "GetClasses" для получения коллекции классов - объект типа CoClasses
[+] новый метод плагина "GetDefFiles" для получения коллекции файлов-определений

[-] Исправлен баг: если def-файл, определенный внутри конфигурации,  уже был открыт ранее, его нельзя было заново открыть/активировать из панели плагина;
[-] Исправлен баг: если модуль класса, определенного внутри конфигурации, уже был открыт ранее, его нельзя было заново открыть/активировать из панели плагина;
[-] Исправлен баг: при открытии из панели плагина класса, определенного внутри конфигурации, выдавалась ненужная ошибка пользователю;
[-] Исправлен баг: при показе синтакс-ошибки внутри класса в сообщении об ошибке указывался неверный номер строки класса;
[-] Исправлен баг - если открыть модуль класса, в котором содержится реализация несколько классов (через //# ClassBegin - //# ClassEnd)
		и выполнить обновление дерева классов, то в дереве в каждом классе из этого .ert показывает методы ВС
		ЕХ классов из этого модуля.
		http://www.1cpp.ru/forum/YaBB.pl?num=1159780896/25#25

1.1.0.0
[+] Важно: плагин Visual1C++ превращен в ActiveX-плагин для Опенконф
для возможности программного управления плагином
[+] пока определен один макрос RefreshAll
[+] Новое свойство плагина - FlagGroupByType = 1 или 0
[+] добавлена раскраска предопределенных функций вирт() и я()
[-] Исправлен баг при одновременном чтении файлов описаний и определений классов разными пользователями
Код сделан аналогично 1С++, где баг давно решен.
[+] Плагин научился работать с классами, расположенными в длл-файлах

1.0.0.14
[+] Плагин Visual1C++ научился правильно проводить синтакс-контроль для классов, содержащих вызовы предопределенных функций КОП вирт() и я()
[http://cvs.alterplast.ru/bugs/show_bug.cgi?id=3889]

1.0.0.13
[-] исправлен баг с двойной проверкой при полном синтакс-контроле

1.0.0.12
[+] добавлена возможность нормального синтакс-контроля
	для классов, определенных в одном файле с помощью директив
	//# ClassBegin и //# ClassEnd
[-] неверно работала синтакс-проверка всех классов,
	если есть классы, определенные в одном файле
[-] убраны ненужные повторящиеся сообщения при полной проверке
		всех классов или всей конфигурации 1С
[-] Исправлено падение Конфигуратора при попытке просмотра свойст
		в в Визуал1С++

1.0.0.11
[+] Добавлена поддержка обычной синтакс-проверки любых классов
	(как в ert-файлах, так и в текстовых файлах)

1.0.0.10
[+] Добавлен драг-дроп в дереве. Теперь можно таскать в текст название
классов, переменные, названия методов.

[-] Исправлена ошибка при работе с именами файлов-классов в кавычках,
например,
class КОП_ДляТестыИсходнойТаблицы = "ExtForms\ТестыИсходнойТаблицы.ert"

[+] Решение бага 2291 [http://cvs.alterplast.ru/bugs/show_bug.cgi?id=2291]
	Исправлен баг с падением 1С, если использована зацикленная иерархия классов

[+] Решение бага 1605 [http://cvs.alterplast.ru/bugs/show_bug.cgi?id=1605]
	1. появилась возможность задавать //include defcls.prm
	относительно папки текущего разбираемого defcls.prm;
	Если задан относительный путь, он сначала проверяется относительно каталога ИБ,
	а уже затем относительно папки текущего разбираемого defcls.prm;
	2. появилась возможность задавать в пути к файлу класса или defcls.prm в //#include
	относительные пути с использованием "." или "..";
